# Data Model: PVC Configuration

## PVC Configuration Variables

These variables are stored in the service's `.env` file and consumed by `init.sh` to render k8s templates.

| Variable | Type | Default | Description | Validation |
|----------|------|---------|-------------|------------|
| PVC | boolean (string) | `false` | Master toggle — must be exactly `true` to enable PVC generation | Must be `true` or `false` (case-sensitive) |
| PVC_NAME | string | `<app-name>-data` | Name of the PersistentVolumeClaim resource | K8s naming: lowercase, hyphens, max 253 chars |
| PVC_MOUNT_PATH | string | `/data` | Container filesystem path where volume is mounted | Absolute path starting with `/` |
| PVC_SIZE | string | `1Gi` | Requested storage capacity | K8s resource quantity (e.g., `1Gi`, `500Mi`, `10Gi`) |
| PVC_ACCESS_MODE | string | `ReadWriteOnce` | PVC access mode | Must be one of: `ReadWriteOnce`, `ReadOnlyMany`, `ReadWriteMany` |
| PVC_STORAGE_CLASS | string | `standard` | Storage class name | Must match a StorageClass in the target cluster |

## Variable Derivation Rules

- `PVC_NAME` defaults to `${APP_NAME}-data` when not explicitly set
- All PVC variables are only written to `.env` when `PVC=true`
- If `.env` already contains PVC variables, their values are preserved on re-scaffold (user customizations respected)
- If `PVC=true` but `PVC_NAME`/`PVC_MOUNT_PATH` are missing from `.env`, defaults are applied

## State Transitions

### PVC toggle transition

```
[PVC unset or false] ──set PVC=true──> [PVC=true: PVC vars populated, PVC manifest generated]
                                          │
                                          │ set PVC=false (--force)
                                          ▼
                                    [PVC vars removed from .env, PVC manifest removed]

                                          │
                                          │ set PVC=false (no --force)
                                          ▼
                                    [WARNING displayed, PVC manifest kept, PVC vars kept in .env]
```

### Re-scaffold behavior

```
[Existing .env with PVC=true + custom PVC vars]
                    │
                    v
[init.sh reads .env → preserves custom PVC vars → re-writes .env]
[Generates PVC manifest using preserved values]
```

## Template Variables

These variables flow into the k8s tmpl.yaml files:

| Template Var | Source | Used In |
|-------------|--------|---------|
| `${VOLUME_MOUNTS}` | Generated by init.sh when PVC=true | deploy.tmpl.yaml (container spec) |
| `${VOLUMES}` | Generated by init.sh when PVC=true | deploy.tmpl.yaml (pod spec) |
| App-specific PVC vars (PVC_NAME, etc.) | pvc.tmpl.yaml | pvc.tmpl.yaml |
